==================================================
HARDCODE REMOVAL REQUIREMENTS
==================================================

Identify and remove unnecessary hardcoded values across the project.

Check for hardcoded:

- API URLs
- user IDs
- restaurant IDs
- branch IDs
- role names
- order statuses
- payment methods
- tax values
- service charge values
- currency formatting
- inventory thresholds
- category names
- menu item names
- dashboard metrics
- dummy analytics data
- static order data
- static inventory data
- static employee data
- repeated labels
- repeated route paths
- repeated status colors
- repeated permission logic
- repeated error messages

Replace hardcoded values with:

- constants
- enums
- config files
- environment variables
- database values
- reusable helper functions
- centralized permission helpers
- centralized status mapping
- centralized formatting utilities

Required examples:

1. Order status labels must use centralized constants.

2. Payment methods must use constants or database configuration.

3. Tax and service charge must come from restaurant settings, not fixed numbers in the code.

4. Currency formatting must use a reusable utility function.

5. Dashboard analytics must come from database queries, not dummy static data.

6. Inventory low-stock threshold must come from config or database setting.

7. API response messages should be consistent and reusable where possible.

8. Role permission logic must not be repeated inside every component or API route.

9. Route paths should be centralized when reused.

10. Status badge colors must come from centralized status color mapping.

Important rules:

- Do not remove necessary mock data if it is clearly used only for development preview.
- If mock data exists, move it to a dedicated mock file.
- Do not randomly delete data without checking usage.
- Do not break existing UI while replacing hardcoded values.
- Replace hardcoded values incrementally and safely.
- Explain every hardcoded value that was removed.
- Explain where the new reusable value is located.

After removing hardcoded values, provide:

1. List of hardcoded values found
2. Files affected
3. Replacement strategy
4. New constants/config/helpers created
5. Remaining hardcoded values that should be fixed later
